# Настройка HTTPS на Nginx с reverse proxy для backend-приложения

## Обзор
В этой инструкции вы настроите Nginx в качестве **TLS-терминирующего reverse proxy** для backend-приложения, работающего на порту `:8080`. Вы создадите самоподписанный сертификат, настроите server block с HTTPS и проверите работоспособность. Документация ведётся в Git (Docs‑as‑Code).

```
sequenceDiagram
participant Client as Клиент (браузер/curl)
participant Nginx as Nginx (порт 443)
participant Backend as Backend (порт 8080)
Client->>Nginx: HTTPS-запрос
Nginx-->>Client: TLS handshake (самоподписанный сертификат)
Nginx->>Backend: proxy\_pass http://127.0.0.1:8080
Backend-->>Nginx: HTTP-ответ
Nginx-->>Client: Зашифрованный ответ
```

## Предварительные требования (Prerequisites)

| Требование | Проверка / команда |
|------------|---------------------|
| Сервер Ubuntu 22.04 / 24.04 (или Debian 12) | `lsb\_release -a` |
| Доступ к терминалу с правами `sudo` | `sudo -v` |
| Backend-приложение слушает `127.0.0.1:8080` | `curl -I http://127.0.0.1:8080` |
| Установлен Git | `git --version` |
| Открытый порт 443 в файрволе | `sudo ufw allow 443/tcp` |

**Примечание.** Для production используйте сертификаты от доверенного CA (Let’s Encrypt). Самоподписанные — только для тестов и разработки.**

## Пошаговая инструкция
### 1. Установка Nginx

```
sudo apt update
sudo apt install -y nginx
sudo systemctl enable --now nginx
```

Проверьте работу HTTP по умолчанию:
```
curl -I http://localhost
```

[Screenshot: `nginx-default-page.png` – стандартная приветственная страница Nginx]

### 2. Генерация самоподписанного сертификата (OpenSSL)
Создайте каталог для сертификатов и сгенерируйте ключ + сертификат:

```
sudo mkdir -p /etc/nginx/ssl
cd /etc/nginx/ssl
sudo openssl req -x509 -nodes -days 365 -newkey rsa:2048 \
-keyout selfsigned.key -out selfsigned.crt \
-subj "/CN=localhost"
```

| Опция | Назначение |
|-------|-------------|
| `-x509` | Создаёт самоподписной сертификат (вместо запроса на подпись CSR) |
| `-nodes` | Не шифрует приватный ключ (без пароля) |
| `-days 365` | Срок действия 1 год |
| `-newkey rsa:2048` | Генерирует новую пару RSA 2048 бит |
| `-subj "/CN=localhost"` | Common Name = localhost (для тестов) |

[Screenshot: `openssl-generate.png` – вывод команды `openssl req`]

### 3. Настройка server block с SSL и reverse proxy
Создайте файл конфигурации:

```
sudo nano /etc/nginx/sites-available/backend\_proxy
```

Вставьте следующую конфигурацию:
```
server {
listen 443 ssl;
server\_name localhost;
ssl\_certificate     /etc/nginx/ssl/selfsigned.crt;
ssl\_certificate\_key /etc/nginx/ssl/selfsigned.key;
# Рекомендованные настройки TLS (для самоподписанного можно упростить)
ssl\_protocols TLSv1.2 TLSv1.3;
ssl\_ciphers HIGH:!aNULL:!MD5;
location / {
proxy\_pass http://127.0.0.1:8080;
proxy\_set\_header Host $host;
proxy\_set\_header X-Real-IP $remote\_addr;
proxy\_set\_header X-Forwarded-For $proxy\_add\_x\_forwarded\_for;
proxy\_set\_header X-Forwarded-Proto $scheme;
}
}
```

**Активация сайта и проверка синтаксиса:**
```
sudo ln -s /etc/nginx/sites-available/backend\_proxy /etc/nginx/sites-enabled/
sudo nginx -t   # проверить конфигурацию
sudo systemctl reload nginx
```

#### Таблица ключевых параметров `proxy\_pass`
| Параметр | Значение | Пояснение |
|----------|----------|-----------|
| `proxy\_pass` | `http://127.0.0.1:8080` | Адрес backend (без HTTPS внутри Docker/локально) |
| `proxy\_set\_header Host $host` | Передаёт оригинальный заголовок Host |
| `proxy\_set\_header X-Forwarded-Proto https` | Указывает backend, что исходный протокол — HTTPS |
| `proxy\_set\_header X-Real-IP` | Реальный IP клиента |

[Screenshot: `nginx-config-editor.png` – содержимое файла `backend\_proxy` в nano]

### 4. Проверка работоспособности
Убедитесь, что backend (например, простой HTTP-сервер) запущен на порту 8080:

```
# Запустите тестовый backend на 8080 (однострочник на Python)
python3 -m http.server 8080 &
```

Выполните проверку через HTTPS с игнорированием самоподписного сертификата:
```
curl -k https://localhost
```

**Ожидаемый результат:** содержимое каталога, отдаваемое Python-сервером (или ответ вашего реального backend-приложения).

[Screenshot: `curl-success.png` – вывод `curl -k https://localhost`]

Откройте в браузере: `https://localhost`. Браузер покажет предупреждение (самоподписной сертификат). Примите риск и убедитесь, что вы видите ответ вашего backend.

## Диагностика и устранение неисправностей (Troubleshooting)

| Проблема | Вероятная причина | Решение |
|----------|-------------------|---------|
| **`nginx: [emerg] ... unknown directive "ssl"`** | Nginx собран без модуля HTTP SSL (редко) или нет пакета `nginx-full`. | Установите `sudo apt install nginx-full`. Проверьте: `nginx -V 2>&1 \| grep http\_ssl\_module` |
| **`SSL\_ERROR\_UNSUPPORTED\_VERSION` в браузере** | Клиент и сервер не согласовали версию TLS. | Укажите в конфигурации `ssl\_protocols TLSv1.2 TLSv1.3;` и перезагрузите Nginx. |**
| **502 Bad Gateway** | Backend не отвечает на `127.0.0.1:8080` или нет `proxy\_pass`. | Проверьте: `curl http://127.0.0.1:8080` должен работать локально. Посмотрите логи: `sudo tail -f /var/log/nginx/error.log` |
